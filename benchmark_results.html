LLM Query Parsing Accuracy by Model Size

Tests across 80 queries covering file type, temporal, combined, and noise-resistance categories.
Each model runs the identical test suite against a llama-server endpoint, aimed at testing smol models (<3b) to support potato devices.

Model Results

Model Params Overall File Type Temporal Specificity

Overall averages all categories into a single score. File Type tests whether the model correctly maps descriptions (e.g., “python scripts”) to the right extension (.py). Temporal evaluates parsing of time expressions (“3 days ago”, “last week”) into structured date ranges. Specificity measures the model’s ability to determine whether a query refers to a specific file or a broad category.
